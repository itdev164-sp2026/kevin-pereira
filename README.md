SC Internacional Website – Requirements Document
1. Project Description

This project is a fan website about Sport Club Internacional, a Brazilian football club based in Porto Alegre, Brazil. The purpose of the website is to provide visitors with information about the club, including its history, featured players, upcoming matches, and recent news.

The website is built using React and Next.js, styled with Styled Components, and prepared to connect with Contentful, a headless CMS that allows content such as news articles or player profiles to be updated without modifying the source code.


2. Feature List

The website contains the following main features:

Navigation Bar
Displays the site title and club identity at the top of the page.

Hero Section
A main banner introducing SC Internacional.

About Section
Provides general information about the club.

Club History Section
Displays background information and major historical context about SC Internacional.

Featured Players Section
Highlights legendary or important players associated with the club.

Match Center Section
Displays upcoming matches or important fixtures.

Latest News Section
Displays recent news related to the club.

Footer Section
Displays site credits and technology used.


3. User Stories / Tasks
Navigation Bar

As a visitor, I want a clear navigation header so I know what site I am visiting.

Hero Section

As a visitor, I want a visually clear introduction to the club when I enter the site.

About Section

As a visitor, I want to read basic information about SC Internacional so I can learn about the club.

Club History

As a fan, I want to learn about the club’s history so I understand its legacy.

Featured Players

As a visitor, I want to see important players so I can recognize key figures in the club.

Match Center

As a fan, I want to see upcoming matches so I know when the team will play.

News Section

As a visitor, I want to read recent updates about the club.

CMS Integration

As a content manager, I want to update news or player information through a CMS without editing the code.
